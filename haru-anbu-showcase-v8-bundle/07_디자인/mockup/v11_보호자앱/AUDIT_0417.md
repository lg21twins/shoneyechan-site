## 4. 공통 이슈 Top 5 (앱 전반)

### 4-1. 🟥 "AI가 만든 느낌" — 제너릭한 아이콘/카피/여백

**증상**
- 모든 카드가 "아이콘 + 제목 + 설명 + 배지" 표준 패턴
- placeholder 텍스트가 매우 사무적 ("메시지 입력", "검색")
- 아이콘 색상이 페이지마다 발산 (파랑/초록/황색/빨강)

**해결**
- **아이콘 색상 시스템 재정의**: 기능 의미 기반 5색만 사용 (주황/파랑/초록/황색/빨강) — 나머지는 무채색
- **placeholder 정서화**: "메시지 입력" → "어머님께 전할 말씀을 적어주세요"
- **카피 1패스 리라이트**: 명사/기능어 → 동사/감정어

### 4-2. 🟥 정서적 거리감 — 관리 도구 ↔ 돌봄 플랫폼

**증상**
- "케어 대상", "담당 간호사", "입소 경과" 같은 **사무 용어**가 중심
- 보호자가 "멀리서 걱정하는 가족"이란 감정을 UI가 이해하지 못함
- 긍정적 순간("오늘 잘 지내셨어요")의 시각적 강조 없음

**해결**
- **"오늘의 좋았던 순간" 섹션** 추가 (홈 상단 또는 리포트 첫 줄)
- 모든 수치에 **감정 라벨** 병기: "58% (평소보다 좋아요)", "기분 87 (웃음 많은 하루)"
- 간호사/가족의 첫 메시지를 **템플릿으로 따뜻하게 고정** (4-3 참고)

### 4-3. 🟧 빈 상태·로딩·에러 UI 부재

**증상**
- 데이터 0건일 때 빈 카드만 표시
- 로딩 스피너/스켈레톤 없음
- 네트워크 에러, 권한 거부 등 에러 상태 화면 없음

**해결** (공모전용 최소한)
- **공통 EmptyState 컴포넌트**: 일러스트 + "아직 기록이 없어요" + 액션
- **Skeleton shimmer**: 채팅/타임라인/리포트 3곳만이라도
- SOS 실패 시 **재시도 버튼** 명시 (생명과 직결)

### 4-4. 🟧 터치타겟 · 접근성 구멍

**증상**
- 세그먼트 탭, 필터 칩 일부가 44pt 미만
- 포커스링 커스텀 스타일 없음 (키보드 사용자 불편)
- 아이콘 전용 버튼에 aria-label 누락

**해결**
- 모든 인터랙션 요소 `min-height: 44px` 강제
- `:focus-visible` 스타일 글로벌 정의 (2px #2C7AFC outline)
- 아이콘 버튼에 aria-label 일괄 추가

### 4-5. 🟨 마이크로 인터랙션 부족

**증상**
- 탭/누름 피드백이 약함 (투명도 변화 미미)
- "근무중", "확인 필요" 등 상태 배지가 **정적**
- 페이지 전환 시 연속성 없음

**해결**
- 모든 탭 가능 카드에 **press-state scale(0.98)** + 150ms
- "근무 중" 초록점 **pulse 2s infinite** 애니메이션
- 중요 알림 배지 **1회 bounce** 진입 애니메이션

## 5. 임팩트 큰 개선 3가지 (공모전 결정타)

### ⭐ 개선 1. 홈 인사말을 "스토리"로 (비용 대비 효과 최고)

**현재**
```
편안한 밤 되세요, 정희님도 잘 쉬고 있어요
오늘 하루도 마무리했어요.
```

**After**
```
편안한 밤 되세요, 예찬님.
정희님은 오늘 산책 15분으로 하루를 시작하셨고,
저녁 식사는 평소보다 잘 드셨어요. 지금은 편히 주무시고 계세요.
```

- 시간대(아침/점심/저녁/밤)별로 **오늘 있었던 3가지 사실**을 자동 구성
- 더미 데이터라도 의미 있는 템플릿 설계
- 심사위원이 가장 먼저 보는 첫 화면 → **"이 앱은 가족을 이해한다"는 신호**

**구현 난이도**: ★☆☆ (템플릿 + JS 문자열 조합)

---

### ⭐ 개선 2. "오늘의 이야기" — 자정 자동 생성 AI 일기

새로운 미니 페이지 (홈 카드에서 진입):

```
📓 정희님의 4월 17일

좋았던 순간 ☀️
"오전 산책 때 햇빛이 좋아 오랫동안 머무르셨어요."

함께 봐야 할 점 🔍
"저녁 투약 시간이 30분 늦어져서 다음부터 조정 예정이에요."

간호사님의 한마디 💬
"오늘 기분이 정말 좋아 보이셨어요. 가족분께도 전해드릴게요."

[➕ 가족에게 공유]  [📝 메모 남기기]
```

**왜 심사위원이 "와" 할까**
- 요양원 앱들 중 **"감정의 기록"을 다루는 제품 거의 없음**
- 단순 수치가 아니라 **서사**로 전달 → 복지·돌봄 공모전 평가 기준에 직결
- 시니어 가족의 죄책감을 덜어주는 설계 (멀리서도 함께 있음)

**구현 난이도**: ★★☆ (새 페이지 1개 + 템플릿)

---

### ⭐ 개선 3. 글로벌 SOS 배너 → "실시간 대응 허브"로 승격

**현재**: 빨간 배너에 "낙상 감지 · 302호 | 2분 전" 정적 표시

**After**:
```
[🔴 펄싱] 낙상 감지 · 302호 · 3분 경과

현재 상황:
김미영 간호사 현장 확인 완료 → 재측정 진행 중
                                   ● ● ● (live)

[📞 즉시 통화]   [자세히 보기 →]

다음 업데이트 예정: 약 5분 후
```

- 배너가 **1분마다 상태 자동 갱신** (Step 1/2/3 시각화)
- **"다음 업데이트 예정 시간"**이 보호자 안심의 핵심
- SOS 진행 중에는 다른 페이지로 이동해도 **미니 바**로 계속 따라다님

**왜 심사위원이 "와" 할까**
- "SOS = 빨간 버튼" 수준의 앱이 대부분 → 우리는 **대응 프로세스를 시각화**
- 실제로 구현 가능한 기능 (타이머 + 상태 머신)
- 공모전의 "사회 임팩트" 점수에 직결

**구현 난이도**: ★★☆ (전역 배너 컴포넌트 확장)
