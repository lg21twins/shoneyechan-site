EVIDENCE 03 · INFORMATION ARCHITECTURE

화면을 그리기 전에 구조를 다시 짰습니다

현재 NASA 앱은 콘텐츠 타입으로 나뉜 5탭(Image/Video/News)입니다. 저는 이걸 사용자 의도 기반 4탭으로 재편했습니다 — Today / Explore / Live / Library, 총 30개 노드. 이건 디자인이 아니라 정보설계고, AI가 절대 자발적으로 하지 않는 작업입니다.

2026.05.07
IA 확정 — 5탭(콘텐츠 타입) → 4탭(사용자 의도)로 재편 결정. "우주 덕후" 페르소나 정의.
2026.05.10
현재 앱 정밀 분석 — NASA 공식 앱 전수 캡처 + 문제점 감사 문서(Current State Audit) 작성.
2026.05.21
디자인 시스템 토큰 통일 — 액센트 #D92906 확정, Pretendard + Space Mono 역할 분리 확립. 활성 HTML 17개 일괄 정비.
2026.05.28
흩어진 NASA 자산 통합 — APOD·Eyes·DSN·Mars Raw 등 메인 앱 밖 11개 "히든 젬"을 새 IA로 흡수. Mission Control 대시보드 신설.
2026.06.03
전수 정리 + 웹 배포 — 파일 구조 정리, 깨진 참조 복구, shonyechan.com 라이브 배포.

핵심: AI는 "NASA 앱 만들어줘" 한 문장에 즉시 답합니다. 저는 한 달간 분석 → 구조 재설계 → 토큰 시스템 → 9번의 탐색 → 3번의 색 결정을 거쳤습니다. 왼쪽 위 보라색 화면이 "AI가 만들면 나오는 것"이고, 그 모든 결정의 합이 오른쪽 앱입니다.

EVIDENCE 04 · REDESIGN (BEFORE / AFTER)

"새로 만든 것"이 아니라 "다시 설계한 것"입니다

AI는 무에서 그럴듯한 화면을 생성합니다. 저는 현존하는 NASA 공식 앱을 전수 분석하고, 그 구조적 문제(카탈로그형 정부 사이트 구조)를 진단한 뒤 다시 설계했습니다. 리디자인은 기준점이 있는 작업이고, 그게 AI 생성물과의 결정적 차이입니다.

BEFORE · 현재 NASA 공식 앱
진단: 5탭 콘텐츠 카탈로그. 평면 그리드에 썸네일이 나열될 뿐, NASA가 가진 압도적 콘텐츠 자산(APOD 30년치·실시간 ISS·화성 사진)이 묻혀 있음. "지금" 감각도, 몰입도 없음.
AFTER · 리뉴얼
처방: 풀블리드 시네마틱 + 큐레이티드 피드. 사용자 의도 기반 4탭으로 재편하고, 흩어진 NASA 자산을 메인으로 끌어옴. 카탈로그 → 매일 찾아오는 데일리 의식으로 전환.
EVIDENCE 05 · SYSTEM CONSISTENCY

한 화면 운빨이 아니라, 4개 탭이 같은 언어를 씁니다

AI는 화면 하나를 뽑습니다. 두 번째 화면을 시키면 톤이 어긋나죠. 제 앱은 4개 메인 탭 + 서브페이지까지 8개 화면이 동일한 토큰·아이콘·섹션 헤더·간격 시스템을 공유합니다. 아래 네 화면을 나란히 보면 — 같은 손에서 나온 게 보입니다.
